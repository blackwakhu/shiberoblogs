My Beginner guide to NGINX

Derrick Shibero Wakhu September 1, 2025

One important knowledge a web developer must have is the ability to be able to manage their program from the server. The best known and widely used server is the localhost which hosts the programmers web application during the construction phase. For today most of this tasks have been taken over by third party solutions like AWS and Azure from Amazon and Microsoft respectively. With this blog I am going to try to install Nginx and host a simple static website with it from my own machine. 

Installation in Fedora

The operating system that I am going to use is Fedora Linux. This is my daily driver. The Nginx package is present in the dnf package manager. Run the following in the terminal.

sudo dnf install nginx

To be able to start and enable run the following commands. The enable commands allows the user not to start the instance of Nginx when they reboot the system.

# start
sudo systemctl start nginx

# enable
sudo systemctl enable nginx

# to stop
sudo systemctl stop nginx

# restart
sudo systemctl restart nginx

Nginx by default is usually on port 80. Open the browser and type on the header http://127.0.0.1:80. The following page will be displayed

For me the preferred port for Nginx is port 90. This is as there is Apache already installed in my system and in order to avoid conflicts. 

Adding the Static File

In order to do this, we are going to alter the Nginx configuration file. This is located in the /etc/nginx folder in Fedora. Navigate to this folder and open the nginx.conf file using your preferred text editor. For me its Neovim. Be sure to have super user rights. Add the following configurations to that file.

http {
    include mime.types; # this will prevent the css errors of text/plain
    server {
        listen 80; # the port to listen to. 
        location / {
            root /usr/share/nginx/html; # the folder containing the static pages to use
         }
    }
}

events {
}

We are going to add a HTML file with the following code. This will replace the index.html file in the /usr/share/nginx/html folder. 

<!DOCTYPE html>
<html>
  <head>
    <title>Welcome to Nginx</title>
     <link rel="stylesheet" href="./style.css"/>
  </head>
  <body>
    <h1>Hello World, this is nginx</h1>
  </body>
</html>

Lets add CSS code that beautifies the above by giving the background a yellow background in a style.css file in the same folder.

body {
   background: yellow;
}
